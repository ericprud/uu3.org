Harmonizing data with domain ontologies

Overall Goals

    Provide consistent view of data across <you're three-letter pharmaceutical here> to:

  1. share common-sense models
  2. refined by user experience (as simple as possible but no simpler)
  3. manifest common coding styles (e.g. what ontologies to use)
  4. provide nucleation site for expanding KR
  5. collaborate on knowledge representation
  6. give experienced people a leg when they cross domains

Topics modeled

  1. Recipe libraries - common components for multiple products, e.g. small molecule library
  2. Process Flow recipes - specific to a product, e g. Menin inhibitor
  3. Recipe execution - manufacturing data capture for quality assurance and analytics
  4. Criticality report - analysis of a campaing for a particular product

Technologies

  1. RDF - graph-based data representation meta-model
  2. SPARQL - query language for RDF
  3. ShEx and SHACL - schema languages for RDF
  4. Graph stores - places to store and query the RDF
  5. Graph virtualization - views of other technologies (SQL) in RDF

Re-used Ontologies

  1. IDMP (Identification of Medicinal Products) - Pistoia Alliance product
  2. cmc-sg - JnJ-specific ontology for expressing CMC data
  3. gist - intra-industry ontology for enterprise data
  4. prov - W3C Provenance Ontology
  5. time - W3C onotology for expressing time intervals
  6. ~10 task-specific specific ontologies

Developed Ontologies

  1. pf - Process Flow-specific structures
  2. exec - CMC execution structures
  3. DPDS-SM - small molecule recipe library

Process

  1. In depth of domain subject
  2. Isolation of entity types (via function properties)
  3. Lexical search on Web (or memory) for entites
  4. Correlation of discovered ontologies with entity attributes
  5. Invention of types and attributes for remaining data
